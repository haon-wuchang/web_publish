1. 리액트 란 ?💥
 - 사용자 인터페이스를 만들기 위한 JavaScript 라이브러리
    사용자 인터페이스(user interface=ui)
 - Meta( 구 페이스북)에서 2013 오픈소스로 공개 하였다, 2015년도에 리액트네이티브가 출시되었다
    리액트네이티브는 앱 만드는거임
    리액트는 홈페이지 만드는거
 - 2019년도에 리액트를 함수형태로 수정하여 업데이트 되었따. 그전에는 클래스 기반이엇다
 - 2022년 서버사이드랜더링이 가능해졌다

2. 리액트의 장점💥
 - 빠른 업데이트와 랜더링 속도 : 이유 =React DOM을 활용하기때문에 ㄱㄴ 하다
 - 컴포넌트 기반 구조 : 함수기반의 컴포넌트 생성
    컴포넌트는 ui를 만들어주는 하나의 블럭이라고 생각행
 - 재 사용성 : 이 가능한이유는 함수기반의 컴포넌트이기 때문이당
 - 든든한 지원군 : Meta( 구 페이스북) 에서 지속적으로 업데이트하기때문에 리액트가 쉽게 사라지진 않는다(얀 이라는사이트)
 - 활발한 지식공유 & 방대한 커뮤니티(NPM) 
 - 모바일 앱 개발 가능 : 2015년 리액트네이티브 출시 (근데 얘는 잘 안쓴댕)

3. 리액트의 단점
 - 방대한 학습량 : HTML, CSS , 자바스크립트 기본 + react 문법   을 전부 알아야한다
 - 높은 상태 관리 복잡도 : react Hooks 를 통한 상태 관리
 
4. react 프로젝트 생성 및 실행 (리액트프로젝트 = 리엑트앱)
    1. node js 설치가 선행되어져야한다   node -v 로 버전확인하기
    2. NPM 커뮤니티를 통해 React앱을 생성함
    3. React앱 생성하기
     -> npx create-react-app 생성할 리액트앱 이름💥  
        -> 생성할 리액트앱이름은 스네이크타입으로 줘야함 (카멜케이스 x)
        -> 만들때는 터미널 안에서 생성해줘야한다 !(vscode에서 터미널말고 깃배시나 cmd 에서 )
        -> 생성되면 vscode 에 haon-app 이 하나 만들어지는데 여기에 하위파일들의 내용/제목/확장자/ id명root 는 절대 건들이면 안된다
    4. 리액트앱 실행(깃배시나 cmd 에서 )
     -> 명령어 💥 
      cd 이동할리액트앱이름(haon-app)
      haon-app> npm start
      -> 이케하면 웹브라우저가 하나 띄워지게됨(로컬호스트3000번)
      -> 웹브라우저 화면 종료하기 : (깃배시나 cmd 에서) ctrl+c 누르고 Y 누르면 종료가 된다

5. 리액트 실행 순서💥
   1. npm start -> src폴더의 index.js 실행 -> App.js 실행
   -> 화면에 보여지는 컴포넌트의 이름은 첫글자 대문자로 적어야한다(ex. App.js)
   -> App.js 대신 내가 만든 파일을 넣을 수 있따.(App2.jsx)


npm start 를 실행하면 src폴더의 그다음에 index.js 를 실행한다 그 안에서  App.js를 실행한다

- 퍼블릭폴더에는 index.html 과 로고이미지가 들어있다
- src 폴더에는 css파일과 js 파일이 들어있다

6.  컴포넌트 (conponent)
   - 리액트의 UI를 구성하는 함수 컴포넌트 
   - 첫글자는 대문자로 정의한다, 카멜케이스로 
   - return 키워드 다움에는 ( ) 로 묶어준다 / { } x
   - 반환되는 컴포넌트는 반드시 하나의 태그로 그룹핑을 한 후 반환하도록 한다💥
      -> 예) <div>, <> 빈태그 로 그룹핑을 해준다
   - 로직을 실행하는 코드 혹은 데이터는 {} 로 감싼 후 반환코드에 추가한다
